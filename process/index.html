De hipóteses a produtos

Numa startup, aprendizado validado é a medida de sucesso. É este aprendizado que indica o quão replicável e escalável será modelo de negócio. Com a experiência de quem já empreendeu, criamos um processo que, seguindo as premissas do Lean Startup, permite que suas hipóteses sejam transformadas em produtos e, muito mais importante, em aprendizado para o seu negócio. O seu produto será construído de forma incremental. Isso quer dizer que entregaremos uma versão funcional após um período curto de desenvolvimento. Dessa forma estamos preparados para mudanças de escopo e objetivos.

Metodologia

  1. Análise do negócio

    Esta fase tem como objetivo identificar e entender o objetivo geral do projeto, e criar um alinhamento entre o cliente e a equipe de desenvolvimento. Além disso, esta será a primeira oportunidade para, dado o objetivo macro, possibilitar a priorização de objetivos específicos.

    • Roadmap

  2. Planejamento

    Acontece no início da iteração e é time-boxed. Nesta ocasião a equipe define o escopo das tarefas e estima o esforço necessário para executá-las.

    O escopo, além de servir de suporte para a estimativa, sintetiza para o cliente as questões em aberto e a dimensão de uma funcionalidade. As estimativas permitem que o cliente possua uma medida relativa sobre a complexidade das tarefas além de ser possível prever quanto tempo elas levarão para serem criadas.

    • Backlog funcionalidades detalhadas e estimadas

  3. Proposta de interface gráfica

    É nesta fase que acontece o ciclo de design conforme descrito em “Como fazemos Design?”. O Designer de Interação, colaborando com clientes e usuários finais valida e define métricas para a funcionalidade.

    Validação com usuários do sistema é uma atividade muito essencial para ser deixada para o fim do processo. Aqui serão utilizadas abordagens integrativas para combinar insights e ideias numa solução.

    Um conceito que é importante para startups é o de aprendizado validado. É importante que os empreendedores sejam capazes de perceber, numa escala pequena, se o modelo proposto é realmente escalável. Uma forma de adquirir este aprendizado é através de métricas. Nesta fase o Designer de Interação, junto com o cliente, irão definir indicativos e métricas para que seja possível medir se a funcionalidade atingiu seu objetivo

    No fim desta fase o Designer de Interação gravará um screencast mostrando uma síntese do racional seguido na criação da funcionalidade.

    • Interface gráfica screencast
    • Métricas

  4. Desenvolvimento

    O nosso time começará da dar vida à proposta à interface gráfica. Durante esta fase, lançaremos uma versão funcional para que você e alguns usuários finais possam testar.

    Utilizamos desenvolvimento orientado a testes, o que faz com que o software esteja preparado para modificações e evoluções constantes.

    No fim desta fase o Engenheiro de Software gravará um screencast mostrando como utilizar a funcionalidade. Você mesmo poderá testa-la em um ambiente controlado.

    • Funcionalidade acessível para testes
    • Screencast
    • Métricas implementadas

  5. Validação

    Com a funcionalidade liberada para o usuário final, ofereceremos ao cliente formas de monitorar as métricas definidas. Estes dados darão insumos e embasamento para melhorias e funcionalidades novas.

    • Dados
